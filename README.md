# Nixme - NixOS Multi-Host Configuration

**A modular, multi-host NixOS configuration using flakes, profiles, and Home Manager.**

This repository manages multiple NixOS machines with a layered architecture that maximizes code reuse while keeping per-host configurations minimal and declarative.

---

## Architecture Overview

```
nixme/
├── profiles/          # Reusable configuration layers
│   ├── common/        # Base system (all hosts)
│   ├── terminal/      # CLI tools (all hosts)
│   ├── desktop/       # GUI environment (laptops/workstations)
│   ├── laptop/        # Laptop-specific (power, touchpad)
│   └── server/        # Server-specific (SSH, monitoring)
│
├── hosts/             # Per-host configurations
│   └── ghost/         # Ghost laptop
│       ├── default.nix        # System imports
│       └── hardware-configuration.nix
│
├── users/             # User definitions
│   └── stefan.nix     # stefan-hacks user
│
├── home/              # Home Manager modules
│   ├── common/        # Base home config
│   ├── terminal/      # Terminal tools (kitty, nvim, etc.)
│   ├── desktop/       # Desktop settings (GNOME, themes)
│   └── users/         # Per-user HM configs
│       └── stefan-hacks.nix
│
├── parts/             # flake-parts modules
├── lib/               # Shared Nix functions
└── scripts/           # Build and test scripts
```

### Profile System

**Profiles** are compositions of NixOS options that can be layered together. Each profile is self-contained and can be imported by any host:

| Profile | Purpose | Imported By |
|---------|---------|-------------|
| `common` | Locale, nix settings, security base, common packages | **All hosts** |
| `terminal` | Shells, CLI tools, development environment | **All hosts** |
| `desktop` | GNOME, audio, fonts, flatpak | Laptops/workstations |
| `laptop` | TLP, power management, touchpad, firmware | Laptops |
| `server` | SSH hardening, fail2ban, cockpit, monitoring | Servers |

**Example host configuration:**

```nix
{ config, pkgs, ... }:

{
  imports = [
    # Hardware (generated by nixos-generate-config)
    ./hardware-configuration.nix

    # Profiles (layer from generic to specific)
    ../../profiles/common      # Every machine
    ../../profiles/terminal    # Every machine
    ../../profiles/desktop     # Laptops only
    ../../profiles/laptop      # Laptops only

    # Users
    ../../users/stefan.nix
  ];

  # Host-specific settings
  networking.hostName = "ghost";
  system.stateVersion = "26.05";
}
```

## Quick Start

### Prerequisites

- NixOS installed
- Git configured with SSH key
- `stefan-hacks` user exists with sudo access

### Initial Setup

```bash
# Clone the repository
git clone git@github.com:stefan-hacks/nixme.git ~/.config/nixme
cd ~/.config/nixme

# Copy hardware configuration (MUST DO before first build)
cp /etc/nixos/hardware-configuration.nix hosts/ghost/

# Test the configuration
./scripts/test-build.sh check

# Apply to system
sudo nixos-rebuild switch --flake .#ghost
```

## Usage

### Testing Changes

```bash
# Quick validation (recommended before every rebuild)
./scripts/test-build.sh check

# Full build test (slower, more thorough)
./scripts/test-build.sh build
```

### Rebuilding the System

```bash
cd ~/.config/nixme

# Build and apply
sudo nixos-rebuild switch --flake .#ghost

# Or just build without applying
sudo nixos-rebuild build --flake .#ghost
```

### Adding a New Host

1. Create host directory:
   ```bash
   mkdir -p hosts/newhost
   ```

2. Create `hosts/newhost/default.nix`:
   ```nix
   { ... }:
   {
     imports = [
       ./hardware-configuration.nix
       ../../profiles/common
       ../../profiles/terminal
       # Add appropriate profiles...
       ../../users/stefan.nix
     ];
     networking.hostName = "newhost";
     system.stateVersion = "26.05";
   }
   ```

3. Generate and copy hardware config:
   ```bash
   # On the new host after NixOS install
   nixos-generate-config --show-hardware-config > /tmp/hardware.nix
   scp /tmp/hardware.nix ghost:/home/stefan-hacks/.config/nixme/hosts/newhost/hardware-configuration.nix
   ```

4. Add host to `flake.nix`:
   ```nix
   ghost = mkSystem { name = "ghost"; system = "x86_64-linux"; };
   newhost = mkSystem { name = "newhost"; system = "x86_64-linux"; };
   ```

5. Test and apply:
   ```bash
   ./scripts/test-build.sh check
   sudo nixos-rebuild switch --flake .#newhost
   ```

## Directory Deep-Dive

### `profiles/`

Each profile is a NixOS module that can be imported. They use `lib.mkDefault` to allow easy overriding.

```nix
# Example: profiles/terminal/default.nix
{ config, pkgs, lib, ... }:
{
  # These can be overridden by hosts
  programs.neovim.enable = lib.mkDefault true;
  programs.bash.enable = lib.mkDefault true;

  environment.systemPackages = with pkgs; [
    git
    ripgrep
    fd
    # ...
  ];
}
```

### `home/`

Home Manager modules follow the same pattern. Each user imports the appropriate modules.

```nix
# home/users/stefan-hacks.nix
{ ... }:
{
  imports = [
    ../common
    ../terminal    # kitty, nvim, tmux
    ../desktop     # GNOME settings, themes
  ];

  home.username = "stefan-hacks";
  home.stateVersion = "26.05";
}
```

### `users/`

User definitions separate user accounts from Home Manager configuration:

```nix
# users/stefan.nix
{ ... }:
{
  users.users."stefan-hacks" = {
    uid = 1000;
    isNormalUser = true;
    extraGroups = [ "wheel" "networkmanager" ];
    openssh.authorizedKeys.keys = [ "..." ];
  };
}
```

## Workflow

### The Golden Path

1. **Edit** - Modify files in appropriate directory
2. **Test** - Run `./scripts/test-build.sh check`
3. **Commit** - Clear message describing what and why
4. **Push** - Backup to remote
5. **Apply** - `sudo nixos-rebuild switch --flake .#ghost`

### Commit Message Format

```
<scope>: <imperative description>

Problem: <what was wrong or what needed to change>
Solution: <how you fixed it>
Tested: <verification method>
```

Examples:
```
feat(profiles/desktop): add flatpak support

Problem: Need to install sandboxed applications
Solution: Enable flatpak with flathub remote
Tested: Built successfully, flatpak command available

fix(hosts/ghost): adjust power profile for AC

Problem: CPU was throttling on AC power
Solution: Set PLATFORM_PROFILE_ON_AC to "performance"
Tested: Verified with tlp-stat
```

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Profiles vs modules | Profiles are composable "distributions" of settings; modules are implementation details |
| Separate `users/` from `home/` | User accounts (system) are separate from user environment (HM) |
| `lib.mkDefault` in profiles | Allows easy per-host overrides without complex option merging |
| Container testing | Catches errors before they reach live systems |
| Hardware configs in git | Required for flakes; copy actual file, don't symlink |
| Flake inputs follow nixpkgs | Home Manager version matches NixOS version for compatibility |

## Troubleshooting

### Build Failures

```bash
# Show detailed trace
sudo nixos-rebuild switch --flake .#ghost --show-trace

# Check option documentation
nixos-option <option.name>

# Search for packages
nix search nixpkgs <term>
```

### Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `attribute 'X' missing` | Package renamed/removed | Search for replacement |
| `option 'Y' renamed` | Breaking change in nixpkgs | Update to new option path |
| `hardware-configuration.nix not found` | Forgot to copy file | `cp /etc/nixos/hardware-configuration.nix hosts/ghost/` |
| `unfree package refused` | Package has unfree license | Set `nixpkgs.config.allowUnfree = true;` or remove package |

### Recovery

```bash
# Boot previous generation from GRUB

# Or rollback from command line
sudo nixos-rebuild switch --rollback

# List generations
sudo nix-env --list-generations --profile /nix/var/nix/profiles/system
```

## Maintenance

### Automatic

```nix
# From profiles/common/default.nix
nix.gc.automatic = true;
nix.gc.dates = "03:00";
nix.gc.options = "--delete-older-than 7d";
```

### Manual

```bash
# Check disk usage
du -sh /nix/store

# Delete old generations
sudo nix-collect-garbage --delete-older-than 3d

# Update dependencies
nix flake update
./scripts/test-build.sh check
```
